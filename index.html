Weird Map Solo

View Live Site | Github


The Portland Weird Map is a Rails app that creates geographical display of "weird" locations around Oregon's biggest city. The Portland Weird Map allows users to quickly become familiar with the locations and descriptions of Portlands most iconic locations. In addition to typical features present in Google Maps API applications (zooming, scrolling, street, and satellite views), the Portland Weird Map allows users to interact with how data points are displayed with category filtering and the ability to add more weird locations. The map contains a Data Map that shows a color coded cluster map of bars in Multnomah County by neighborhood.

Core Competencies: Ruby on Rails, Google Maps Javascript API, GeoJSON

My Work: I built the entire application.

Local Swap Solo

View Live Site | Github


A Ruby on Rails applications designed to connect users who want to trade goods with other users in the same geographic area by allowing users to create profiles in a geographically and socially enabled environment. Cragslist has become a ubiqutious platform and functions effectively although it suffers from a number of scams and quite frankly creepy people. In order to create a safe community of users, Local Swap is only available to authenticated users, and user locations are hidden via a location scrambling function that offsets their given location by up to 500 ft. Directions are available for users to locate other users, but those directions will never point to a user's true location. In this way user locations are used as an asset to the application but not made available for other users.

Core Competencies: Ruby on Rails, PostgreSQL, Authentication using the Devise Gem, Photo Uploads and Image Cropping, Live Chat via ActionCable, Mailboxer Gem messaging and emailing, React-Rails Gem, Geocoder Gem, Google Maps Javascript API, Google Maps Directions API, Seeded data via Faker Gem.

My Work: I built the entire application from end to end, front to back.

Image Manager Pair

View Live Site | Github


A Ruby on Rails applications that makes heavy use of the React-Rails Gem to provide a streamlined experience for users that want a guarded image management platform. Users have the ability to crop and download images that are not saved long term to the database. The cropping process has configurable aspect ratio settings. The Admin user is able to manage saved images through a dashboard built in ReactJS.

Core Competencies: Ruby on Rails, PostgreSQL, Authentication using the Devise Gem, Photo Uploads and Image Cropping, React-Rails Gem, Seeded data via Faker Gem.

My Work: As can be seen from the commit history on the project, I built the all of the react components, the image cropping functionality including the ability to user various aspect ratios, and a substantial portion of the styling. On the admin side of the app, I built the entirety of the table and the form to add new images. I also setup the devise authentication.

Apocalypse App Group

Github


The Apocalypse App is a sinatra with postgresql built application that demonstrates many types of activerecord relationships between its object classes in order to create a turn-based, textually centric game. Users will stock up on resources and see how long they will survive in a given apocalyptic scenario with their given supplies. Random events will be triggered as users progress through the game and at times choices will become available that will serve to either help or hinder the user's progression. User high scores are persisted in the database.

Core Competencies: Sinatra, PostgreSQL, Hazmat Suits

My Work: I wrote the functions allowing for users to move through the "days" that the game is based upon. I wrote the functions surrounding the "special items" required to survive various events, and the functions allowing the store to operate. I created the through database that allows users to have various quantities of different goods and allow that to properly deduct from the user's stockpile of cash. I created the special events that "pop" on certain days forcing the user to make a life or death decision.
